## 任务 2：行业分类树

3 层分类：**一级（行业域）→ 二级（招标/采购子类）→ 三级（触发标签）**。规则的 `conditions.industry` + `project_types` 决定挂载到哪个节点。

```
根
├── 建筑工程（industry: 建筑工程）
│   ├── 施工招标        ← DISC-001 / CAPITAL-001 / CERT-001 / QUAL-001 / PERF-001
│   │   ├── [资质等级]   （施工总承包/专业承包资质、安全生产许可证）
│   │   ├── [业绩要求]   （类似业绩、区域业绩）
│   │   └── [资格门槛]   （注册资本、财务指标）
│   ├── 勘察设计招标    ← 设计资质相关规则（本次未展开）
│   │   ├── [设计资质]
│   │   └── [人员职称]
│   └── 监理招标        ← 监理资质相关规则
│       └── [监理资质]
│
├── 政府采购（industry: 政府采购）
│   ├── 货物采购        ← BRAND-001 / BUDGET-001
│   │   ├── [技术参数]   （品牌/型号/参数限制）
│   │   └── [报价合规]   （预算/最高限价）
│   ├── 服务采购        ← BUDGET-001（服务类报价）
│   │   ├── [服务能力]
│   │   └── [报价合规]
│   └── 工程采购        ← 与建筑工程共享部分资质规则
│       └── [施工资质]
│
├── IT / 信息化（industry: IT）
│   ├── 软件开发采购    ← 软著/CMMI 资质、需求指定品牌检测
│   │   ├── [软件资质]   （软件著作权、CMMI/ISO27001）
│   │   └── [技术锁定]   （指定操作系统/数据库品牌 → 复用 BRAND-001 逻辑）
│   ├── 系统集成采购
│   │   └── [集成资质]   （系统集成企业资质、ITSS）
│   └── 云服务/运维采购
│       └── [等保要求]   （等级保护测评报告）
│
└── 通用（industry: 通用，跨行业）  ← TIME-001 / DEPOSIT-001 / AWARD-001
    ├── 程序时限        （投标准备期 ≥20 日等）
    ├── 保证金          （保证金 ≤2% 估算价）
    └── 定标合规        （中标通知书法律效力）
```

**哪类规则属于哪个行业（挂载逻辑）**：
- **与执行能力强相关、行业专属**的资质/技术/业绩规则挂到具体一级+二级节点（如"施工总承包资质"只挂建筑工程/施工招标，绝不下发到 IT 采购）。
- **纯程序性、跨行业普适**的规则挂"通用"节点（时限、保证金、定标程序——任何行业招标都适用）。
- **同一违规模式在多行业复用**时（如"指定品牌"在货物采购和 IT 软件采购都可能出现），规则逻辑复用，但通过 `industry` + `project_types` 限定触发范围，避免跨行业误报。

**为什么这样分**：`industry` 是防误报的第一道闸门——若不分行业，"建筑工程施工总承包资质"规则会命中 IT 软件采购而 100% 误报。分类树让规则只在自己相关的文档类型上运行。

## 任务 3：边界 Case 分析（看似排斥性 / 实则合规）

以下 3 条表述"长得像"排斥性条款，但属于合理要求。逐条说明规则为何不误报。

### Case 1：「投标人须具备建筑工程施工总承包二级及以上资质」
- **为何像违规**：设置了准入门槛，看似"限制"投标人。
- **为何实则合规**：资质与**合同履行能力直接相关**——没有施工资质就无法合法承接施工，属《招标投标法》允许的资格审查标准（§19）。
- **规则为何不误报**：
  - DISC-001 的 regex 要求出现"注册地/设立机构 + 本市/本省"等**地域要素**，本表述无地域词，不命中。
  - CAPITAL-001 只匹配"注册资本 + 亿/千万"的**畸高资本门槛**，与资质等级无关，不命中。
  - QUAL-001（模糊资质）只在出现"相应/相关资质"**未明确等级**时触发；本表述已明确"施工总承包二级及以上"，反而是 QUAL-001 期望看到的合规写法，不命中。

### Case 2：「投标报价不得超过采购预算 500 万元」
- **为何像违规**：设置了金额上限，像是"限制"。
- **为何实则合规**：这是招标人**依法设定的最高限价/预算**，是合规要求本身（87号令 §51 正是要求超预算作无效投标）。
- **规则为何不误报**：
  - BUDGET-001 是 `field_compare`，比较的是**投标文件里的实际报价** vs 预算；只有当"报价 > 预算"才触发。招标文件中"设定预算上限"这一条款本身不含实际报价数值，不构成 left>right 的比较，不触发。
  - DEPOSIT-001 只针对"投标保证金"字段，与报价预算无关，不命中。

### Case 3：「投标人须提供近三年类似项目业绩不少于 2 个」
- **为何像违规**：业绩门槛看起来在筛人。
- **为何实则合规**：**不限地域**的同类项目业绩，与履约能力相关，属合理资格条件。真正违规的是"本市/本省业绩"这类地域限定（实施条例 §32）。
- **规则为何不误报**：
  - PERF-001 的 regex 强制要求业绩前带**地域词**（本市/本省/本地区/当地）。本表述为"近三年类似项目业绩"，无地域限定词，不命中。
  - 这正是 PERF-001 精心设计的区分点：地域词的有无是"合理业绩要求"与"地域排斥"的分界线。

**共同设计思路**：排斥性规则的 pattern 不匹配"是否设门槛"，而匹配"门槛是否与地域/品牌等履约无关因素挂钩"。合理的资质/业绩/预算要求因缺少这些违规特征词而天然被排除。
